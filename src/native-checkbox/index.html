Cross-browser styling for the native checkboxes.

These custom styles require a specific markup pattern: 

<div class="c-cb">
  <input type="checkbox" name="c_input" id="c_input_1" checked>
  <label for="c_input_1">
    Option 1
  </label>
</div>

The .c-cb container largely serves as the styling hook, and utilizes position: relative; to keep child absolutely positioned elements and pseudo elements, relative to the container.

The actual native checkbox is visually hidden from view. While visually hidden, it is essential for it to remain accessible to screen readers and keyboard focus. Otherwise all the native checkbox functionality would need to be rewritten in JavaScript.

The label's :before and :after pseudo elements are used to recreated the visible checkbox and "check" element.

The styling of the custom checkbox is determined by the state of the actual checkbox. :checked, disabled and :focus will all visually change the styling of the custom checkbox.
