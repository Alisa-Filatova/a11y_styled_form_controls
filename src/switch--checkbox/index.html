Switch Toggle Component: Checkbox

Pattern to restyle a checkbox into a switch component.

See source on GitHub, Index of input patterns

Demo Details

The switch components are coded with the following markup patterns: 

<!-- Version: [type="checkbox"] styled to look like a switch -->
<label class="check-switch" for="switch_1">
  <input type="checkbox" id="switch_1" role="switch">
  <span tabindex="-1">Example Text:</span>
  <!--
    the tabindex="-1" is here to negate VO
    from announcing the "example" label twice
  -->
</label>

<!-- Version: [type="checkbox"][role="switch"] -->
<label class="check-switch" for="switch_2">
  <input type="checkbox" id="switch_2" role="switch">
  <span tabindex="-1">Example Text:</span>
</label>

<!--
  Version: [type="checkbox"][role="switch"]
  Alternate markup/style
-->
<div class="switchbox">
  <label for="switchbox">
    Label Text:
  </label>
  <input type="checkbox" id="switchbox" role="switch">
</div>

Similar to the native checkbox example, a similar pattern utilizing a native checkbox form control can be used to create a custom switch control. However, it should be noted that a form control with the role of switch vs a checkbox role have different expectations to screen reader users.

The native role of a "checkbox" should be used if a switch component acts as a checkbox would. In that the checkbox "switch" should not change the state of a component or a global setting of a site or application, until after a user purposefully submits a form to save this (or any other) change(s). A role="switch" should be applied to the native checkbox if activating the form control will perform an instant state change, without the need for a form submission.

If using a visually styled switch in a website or application, it should adhere to one of the mentioned roles, announcing itself as a switch or a checkbox, and this announcement and the component's behavior should be consistent across all instances of the switch. (For instance, Why would a similar looking UI component require a form submission in some instances, but be an instantaneous change in others?)
