A switch toggle button can be built via one of the following two markup patterns: 

<!-- a toggle button styled to look like a switch -->
<button type="button" id="bg" class="btn-switch" aria-pressed="false">
  <span class="btn-switch__label">
    Label Text
  </span>
  <span class="btn-switch__toggle"></span>
</button>

<!-- progressively enhanced from a checkbox -->
<span data-btn-switch>
  <label for="pe-switch">
    Label Text
  </label>
  <input type="checkbox" id="pe-switch">
</span>

In the context of a control that is meant to perform an instantaneous action, then a button utilizing aria-pressed="true/false" would be an appropriate base for the switch component.

As mentioned in the checkbox pattern example details, checkboxes are primarily used in the context of filling out a form, and they are not typically expected to cause an immediate state change to other components, or the website/app in general, when interacting with them.

On the other hand, activating a button does have a user expectation to cause immediate state change. The usage of aria-pressed="true/false", in combination with a visual state change, will indicate to all users the present state of the button (and its target). A succinct and appropriate button label should provide the context of what the button will toggle.
